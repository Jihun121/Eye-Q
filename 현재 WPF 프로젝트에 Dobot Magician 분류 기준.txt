현재 C# WPF 프로젝트에 Dobot Magician 분류 기능을 통합해줘.

목표 동작은 다음과 같다.

1. Arduino 컨베이어를 13바퀴 이동시킨다.
2. 컨베이어를 정지하고 카메라 위치에서 ONNX 검출을 수행한다.
3. 검출 결과를 Normal / Defect / NoDetection 3가지로 분류한다.
4. NoDetection, 즉 "탐지 없음" 또는 confidence 0.0% 또는 유효 탐지 결과가 없는 경우에는 Dobot을 절대 동작시키지 않는다.
5. Normal 또는 Defect인 경우에만 다시 Arduino 컨베이어를 13바퀴 이동시켜 Dobot Pick 위치까지 보낸다.
6. Dobot Pick 위치에 도착하면 컨베이어를 정지한다.
7. Normal이면 Dobot이 Pick 위치에서 제품을 집고 RightPlace 위치에 놓는다.
8. Defect이면 Dobot이 Pick 위치에서 제품을 집고 LeftPlace 위치에 놓는다.
9. Dobot 동작 중에는 컨베이어가 절대 움직이지 않게 한다.
10. Dobot 분류 명령 후 5초간 대기한 뒤 다음 사이클로 넘어갈 수 있게 한다.

중요한 안전 조건:
- NoDetection일 때는 PickAndPlaceRight(), PickAndPlaceLeft()를 절대 호출하지 말 것.
- if result == Normal else Left 방식으로 구현하지 말 것.
- 반드시 switch 또는 명시적인 if / else if / else 구조로 Normal, Defect, NoDetection을 분리할 것.
- 검출 실패 상태에서 이전 검사 결과를 재사용하지 말 것.
- 매 검사 사이클 시작 시 검사 결과 기본값은 NoDetection으로 초기화할 것.
- Dobot 동작 중 Arduino Move 명령이 실행되지 않게 할 것.

5. Pick 위쪽 위치로 상승

ONNX 결과 처리를 위해 InspectionResult enum을 추가한다.

public enum InspectionResult
{
    Normal,
    Defect,
    NoDetection
}

ONNX 결과 변환 함수를 만들어줘.

조건:
- label이 null, empty, whitespace이면 NoDetection
- label에 "탐지 없음"이 포함되면 NoDetection
- confidence가 0 이하이면 NoDetection
- confidence가 최소 기준값보다 낮으면 NoDetection
- label이 NORMAL 또는 정상 이면 Normal
- label이 FAIL, DEFECT, 불량, SCRATCH, CRACK, CONTAMINATION, EDGE_CHIPPING, PATTERN_DEFECT 등 불량 클래스이면 Defect
- 그 외 알 수 없는 label은 안전하게 NoDetection으로 처리

예시 함수명:
private InspectionResult ConvertToInspectionResult(string label, double confidence)

자동 검사 사이클 함수를 만들어줘.

예시 함수명:
public async Task RunOneInspectionCycleAsync()

흐름:
1. inspectionResult를 NoDetection으로 초기화
2. Arduino 13바퀴 이동
3. 300ms 안정화 대기
4. 카메라 촬영
5. ONNX 추론
6. ConvertToInspectionResult로 Normal / Defect / NoDetection 변환
7. NoDetection이면:
   - 상태 메시지: "탐지 없음: Dobot 동작 안 함. 수동 확인 필요."
   - Arduino StopAsync 호출
   - PickAndPlaceRight / PickAndPlaceLeft 호출 금지
   - return
8. Normal 또는 Defect이면:
   - Arduino 13바퀴 이동
   - 300ms 안정화 대기
   - Normal이면 await Task.Run(() => _dobotService.PickAndPlaceRight())
   - Defect이면 await Task.Run(() => _dobotService.PickAndPlaceLeft())
   - await Task.Delay(5000)
   - 상태 메시지: "분류 완료"

분류 함수는 반드시 이렇게 명시적으로 작성해줘.

switch (inspectionResult)
{
    case InspectionResult.Normal:
        await Task.Run(() => _dobotService.PickAndPlaceRight());
        await Task.Delay(5000);
        break;

    case InspectionResult.Defect:
        await Task.Run(() => _dobotService.PickAndPlaceLeft());
        await Task.Delay(5000);
        break;

    case InspectionResult.NoDetection:
        SetStatus("탐지 없음: Dobot 동작 안 함");
        await _arduinoService.StopAsync();
        return;
}

기존 프로젝트의 Arduino 제어 함수 이름이 다르면, 기존 함수명에 맞게 연결해줘.
예를 들어 기존에 MoveTurnsAsync(13), StopAsync()가 있으면 그대로 사용하고, 없으면 기존 Serial 명령 함수에 맞춰 래핑 함수를 만들어줘.

기존 ONNX 결과 객체가 다를 수 있으므로, 현재 프로젝트의 결과 객체에서 label과 confidence를 가져오는 부분은 실제 코드에 맞춰 수정해줘.
단, UI에 "FAIL [탐지 없음 0.0%]"로 표시되는 경우는 반드시 NoDetection으로 처리해야 한다.

Dobot 연결은 매 사이클마다 반복하지 말고 프로그램 시작 또는 자동 검사 시작 전에 한 번만 연결되게 해줘.
단, 자동 검사 실행 전에 IsConnected가 false이면 Connect()를 한 번 호출해도 된다.

실제 프로젝트에 반영 후 확인할 것:
- DobotDll.dll이 실행 폴더에 복사되는지 확인
- 프로젝트 플랫폼 대상이 테스트 프로젝트와 동일한 x86/x64인지 확인
- COM 포트가 COM7인지 확인
- Dobot Studio, DobotLab, DobotLink가 포트를 점유하지 않는지 확인
- NoDetection 상태에서 Dobot이 절대 움직이지 않는지 확인
- Normal일 때 오른쪽으로 이동하는지 확인
- Defect일 때 왼쪽으로 이동하는지 확인

3. DobotService의 PortName 상수 고정을 제거한다.

현재 DobotService는 private const string PortName = "COM7"; 구조라서 실제 프로젝트에서는 포트를 바꿀 수 없다.
이를 다음 방식 중 하나로 수정한다.

방식 A:
public string Connect(string portName)

방식 B:
생성자에서 portName을 받는다.
public DobotService(string portName)

추천은 방식 A이다.

Connect()를 Connect(string portName)로 바꾸고, 내부에서 선택한 포트명을 사용한다.
연결 성공 후 ConnectedPortName 같은 속성에 저장한다.

예시:
public string? ConnectedPortName { get; private set; }

public string Connect(string portName)
{
    if (IsConnected)
        return $"Already connected: {ConnectedPortName}";

    var result = DobotDll.ConnectDobot(portName, Baudrate, firmwareType, version);

    if (result != NoError)
        throw new InvalidOperationException(GetConnectErrorMessage(result, portName));

    IsConnected = true;
    ConnectedPortName = portName;
}

GetConnectErrorMessage도 고정 PortName이 아니라 portName 파라미터를 받아서 메시지를 만들도록 수정한다.

DobotService에 EmergencyStop() 메서드를 추가한다.

EmergencyStop() 기본 동작:
- 연결되어 있지 않으면 예외 대신 "Dobot not connected" 메시지를 반환하거나 무시한다.
- DobotDll.SetQueuedCmdStopExec() 호출
- DobotDll.SetQueuedCmdClear() 호출
- IsSuctionOn 상태 갱신은 주의한다.
  제품을 들고 있는 중 suction을 강제로 OFF하면 제품이 떨어질 수 있으므로, 기본 EmergencyStop에서는 suction OFF를 자동 호출하지 않는다.
  별도로 사용자가 원하면 Suction OFF 버튼으로 끄게 한다.

예시:
public string EmergencyStop()
{
    if (!IsConnected)
        return "Dobot is not connected.";

    DobotDll.SetQueuedCmdStopExec();
    DobotDll.SetQueuedCmdClear();

    return "Dobot emergency stop executed.";
}

만약 DobotDll.SetQueuedCmdStopExec 함수가 없으면 DobotDll.cs에서 사용 가능한 Queue Stop 관련 함수를 찾아서 적용한다.
사용 가능한 함수명이 다르면 실제 DLL wrapper의 함수명에 맞춰 수정한다.

9. 코드 비하인드 예시를 실제 프로젝트 구조에 맞게 구현한다.

예시:

private void LoadDobotPorts()
{
    var ports = SerialPort.GetPortNames()
        .OrderBy(port => port)
        .ToList();

    DobotPortComboBox.ItemsSource = ports;

    if (ports.Contains("COM7"))
        DobotPortComboBox.SelectedItem = "COM7";
    else if (ports.Count > 0)
        DobotPortComboBox.SelectedIndex = 0;

    DobotConnectButton.IsEnabled = ports.Count > 0;
}

private void RefreshDobotPortsButton_Click(object sender, RoutedEventArgs e)
{
    LoadDobotPorts();
}

private void DobotConnectButton_Click(object sender, RoutedEventArgs e)
{
    try
    {
        var portName = DobotPortComboBox.SelectedItem as string;

        if (string.IsNullOrWhiteSpace(portName))
        {
            DobotStatusText.Text = "Dobot 상태: 포트를 선택하세요.";
            return;
        }

        var message = _dobotService.Connect(portName);
        DobotStatusText.Text = message;
    }
    catch (Exception ex)
    {
        DobotStatusText.Text = $"Dobot 연결 실패: {ex.Message}";
    }
}

private async void DobotEmergencyStopButton_Click(object sender, RoutedEventArgs e)
{
    try
    {
        _isEmergencyStopped = true;

        await _arduinoService.StopAsync();

        var message = _dobotService.EmergencyStop();

        DobotStatusText.Text = $"비상정지: {message}";
    }
    catch (Exception ex)
    {
        DobotStatusText.Text = $"비상정지 오류: {ex.Message}";
    }
}
